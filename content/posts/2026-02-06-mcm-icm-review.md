<div class="article-description" style="text-align: center; font-size: 1.3em; color: #555; margin: 20px 0 30px 0; font-weight: 500;">至少是完赛了吧</div>

没有什么特别的理由，也没有接触过数学建模知识，只是临时被邀请来参加美赛。大概看了一下比赛的玩法，感觉和自己的技术栈还算契合，就同意了。深知这个比赛结果几乎不会对我升学有什么帮助，赛前也是没怎么准备。就大概搜索了一下比赛的大致套路和他人的复盘就直接上了。

我们一开始就是健康比赛的想法，另外两个队友都在学校可以线下，所以第一天睡到了九点多。（虽然第四天弄到了九点多）

吃麦满分的时候在知乎上看到了今年的翻译版。A很显然是某种 ODE 和跨领域的东西，看着人有点慌；B是太空科技话题，疑似多目标优化；C洗数据无聊到一眼抛弃；D不懂体育一眼抛弃；E看着有点意思，但没看懂，值得再看看；F纯扯淡一眼抛弃。

和队友简单讨论和搜索了一两个小时，在中午前后敲定了 B 题，在事后看来是一个不错的选择。因为 A 和 E 跨的领域我们确实不熟悉，而 B 我和另外一位队友学过优化相关的课程，EAP也恰巧写的是 SpaceX 的檄文。


## 时间线 { .section-title }

### Day 1

主要就是讨论（聊天）。B 题是一个极度模糊的题目，进行了一个效率还不错的 brainstorm 和各种假设合理的讨论，敲定了个框架就结束了。由于第一天我和队友都恰巧有其他现实生活中的事情，干的活不多，在情理之内。好消息是精力消耗不大。

### Day 2

最低效的一天。我弄完 assumption 后开始了文章的前半部分的书写和一些模型最根本的优化模型建立。但是队友那边的模型调研一直很低效，讨论到晚上都没结束。由于这一块是后续任务最 base 的数值，所以说没弄完的话后面都推不了。火箭有一块甚至是我实在忍不了了第三天下午被我强行截断接手速战速决的。那天晚上我深知进度严重落后 baseline，自己弄到了凌晨五点把一部分原始资料整合到文章。

### Day 3

勉强步入正轨了。我下午擦完前面的屁股之后在晚上三个小时就结束了 task1。两个队友同时在做更后面 task 的调研和数值建模。不知道是不是有了经验，还是有了我前面文章语料模仿对象的原因，工作质量显著上升了几个档，从无法完赛的预期进度一下子变得有希望了起来。

### Day 4 / 4.5

第四天是一场超级鏖战，我们大约 11 点才开始工作，因为知道今天晚上是要通的所以说多休息了一下。我大约花了三四个小时完成了 task2，花费了一点时间调整了一下前面的所有工作，让它更有连贯性。惊喜的是，我打算开始弄 task3 的时候，发现队友的工作完成度非常高，我只用做微量的调整就能用了。然后就是很顺畅的做了后面的几个 task。大约凌晨两点的时候，文章主体基本搞完了，开始做美工以及写 summary。最后还花了一个小时压缩到 25 页内，然后在八点出头的时候提交了，算是压着 DDL 了。

## 反思 { .section-title }

### 做得好的地方 { .section-title }

- **B 题的 brainstorm 是必要的**，而且我们做的很好，事后看来帮我们省去了不少麻烦。

- **第一天确立了我的专制地位**，也许是我们后期高效的来源。

- **这个比赛要高效应该要线下打**，遇到什么问题直接讨论，不用浪费时间在低效的线上沟通。

- **感谢 cra 提供的免费 overleaf 服务**，让我们不用拘泥于文章版本和嵌合问题。

### 做的不好的地方 { .section-title }

- **前期严重低效**。我觉得有两个问题：第一个问题来源于我，我给的搜索目标太模糊了，导致我想要的东西和队友给的差距很大；第二个问题来源于队友过度依赖于 AI 搜索，而没有筛选好信息和做好合理的组织，浪费了很多时间。

- **美工或者完善论文的时间还是短了**。至少留的时间不足以我释放全部解数，也没有打磨到很完美的状态，我觉得有点可惜。

## 结语 { .section-title }

第一次打美赛，也是最后一次。如果把这当做一次猎奇体验的话，整体还算愉快吧。

完工的时候还是会有一点成就感的。学到什么新东西吧，倒也没有，就是把之前课程、研究积累下来的套路复用一遍而已，完全是存量比拼，最大的收获也许是 leadership 经验...
